一元奇迹测试（v1.0.0）

发表于 2016-10-12   |   分类于 Technology ， Programming Language ， APP   |     |  

登录注册模块

获取验证码：

  1. 点击获取验证码是否可以再点击获取；
  2. 验证码是否发送成功；
  3. 显示验证码发送成功手机是否收到验证码；

用户注册：

  1. 新用户是否可以注册；
  2. 已注册用户是否可以再注册；
  3. 手机号码输入错误是否可注册；
  4. 用户密码输入过短或过长，是否有提示信息；
  5. 用户服务协议是否可查看；
  6. 验证码是否可以正确获取；

手机用户登录：

  1. 手机号格式错误是否提示；
  2. 手机号为空是否提示；
  3. 密码为空是否提示；
  4. 密码错误是否提示；
  5. 密码正确是否正确跳转；

用户修改密码：

  1. 验证码是否正确获取；
  2. 手机号格式错误是否提示；
  3. 密码设置过短或过长是否提示；
  4. 两次密码不一致是否提示；
  5. 提交修改是否显示修改成功；
  6. 用修改的密码重新登录是否可登录；

微信登录：

  1. 微信图标是否显示（若未安装微信则不显示）；
  2. 点击微信是否挂起微信（WeChat）；
  3. 挂起微信是否显示授权信息，授权信息是否正确；
  4. 点击授权是否跳转至绑定手机界面；
  5. 下次微信登录是否直接登录成功（不用再授权绑定手机）；

微信登录绑定手机：

  1. 手机号格式错误是否提示；
  2. 获取验证码是否可以正确获取；
  3. 提交信息是否可以正确提交；
  4. 提交成功是否成功跳转界面；

首页模块

搜索产品：

  1. 点击首页搜索产品是否跳转至搜索界面；
  2. 输入信息搜索是否跳转至搜索结果界面且数据是否正确；
  3. 点击热门搜索是否跳转至搜索结果界面且数据是否正确；
  4. 点击取消是否正确返回；
  5. 搜索结果界面是否可以正确加入购物车；
  6. 加入成功，badge（主界面清单右上角红色数字）是否改变（badge 显示商品种类数，若加入已加入购物车同类商品则不改变）；
  7. 点击商品是否正确跳转至最新一期商品详情界面；

广告栏点击：

  1. 点击广告栏是否跳转至相应界面（商品详情 / 全部商品 / 广告详情网页）；

查看分类：

  1. 点击搜索是否跳转至搜索界面；
  2. 点击相关分类是否跳转至相关数据界面，且数据是否符合分类；
  3. 跳转至的分类商品列表界面标题及商品数量是否正确；
  4. 分类商品列表界面点击“全部加入清单”是否成功；
  5. 分类商品列表界面点击商品是否跳转至最新一期商品详情界面；
  6. 加入成功后 badge（主界面清单右上角红色数字）是否改变；

另立类目（分类旁标签按钮）：

  1. 是否跳转至其应显示的类目商品列表界面；
  2. 商品列表界面标题及商品数量是否正确；
  3. 商品列表界面点击“全部加入清单”是否成功；
  4. 分类商品列表界面点击商品是否跳转至最新一期商品详情界面；
  5. 加入成功后 badge（主界面清单右上角红色数字）是否改变；

查看晒单：

  1. 晒单分享界面数据是否正常显示（时间、用户名、图片数量、商品名、标题、描述）；
  2. 点击图片是否可以正确显示图片；
  3. 查看图片界面图片是否可以正确保存；
  4. 点击用户头像是否跳转至用户个人中心界面；
  5. 点击用户晒单是否跳转至晒单详情界面；
  6. 晒单详情界面数据是否正确显示；
  7. 晒单详情界面点击图片同上；
  8. 点击分享是否可以正确分享；
  9. 打开分享的链接是否有页面显示；

新手指引：

  1. 点击新手指引是否成功跳转至新手指引界面；
  2. 新手指引界面图片是否正确显示；

最新揭晓：

  1. 点击更多是否跳转至主界面“揭晓”栏目；
  2. 揭晓数据是否正常显示；
  3. 揭晓倒计时是否正常刷新；
  4. 点击商品是否跳转至商品详情界面，且该商品为已揭晓或者倒计时状态；

商品展示：

  1. 商品分类排序是否正确；
  2. 商品数据显示是否正确（商品名、商品图、开奖进度）；
  3. 商品数据是否有重复；
  4. 点击进度或者总需人次，排序是否正确颠倒；
  5. 点击商品是否成功跳转至商品详情界面（最新一期）；
  6. 点击加入购物车是否有动画效果；
  7. 加入成功清单 badge（主界面清单右上角红色数字）是否更新；
  8. 连续点击加入购物车 badge 位置是否改变，动画是否卡顿；

其他：

  1. 滚动中奖信息是否正确滚动显示；
  2. 上拉加载是否加载新数据；
  3. 数据已全部加载是否显示“全部加载完毕”；
  4. 下拉刷新是否刷新已有数据；
  5. 界面是否卡顿；

揭晓模块

商品展示：

  1. 揭晓商品数据是否正确显示（商品名、获得用户、揭晓状态等）；
  2. 揭晓状态为倒计时是否正确倒计时；
  3. 点击商品是否跳转至商品详情界面；
  4. 商品详情界面是否正确显示状态（已揭晓 / 倒计时），且数据是否正确；
  5. 倒计时结束揭晓是否及时；

其他：

  1. 界面是否卡顿；
  2. 下拉刷新是否正确刷新数据；
  3. 上拉加载是否正确加载数据；

全部商品模块

分类点击：

  1. 点击左上角分类图标，分类类目是否正确显示，且默认选中全部商品；
  2. 点击商品类目商品列表数据是否刷新，且类目列表隐藏；
  3. 点击不同的类目，商品列表数据是否正确，界面标题是否为该类目；

商品展示：

  1. 商品数据是否正确（商品图、商品名、总需数量、剩余数量、进度）；
  2. 点击商品是否正确跳转至商品详情界面，且为最新一期；
  3. 点击加入购物车是否正确加入；
  4. 加入购物车 badge 是否正确刷新；

清单模块

显示：

  1. 在别的界面加入购物车时是否正确加入；
  2. 多次加入数量是否增加；
  3. 加入购物车的商品数据是否正确显示（商品图、数量、商品名等）；
  4. 当购物车为空时，右上角“编辑 / 完成”是否隐藏，且展示无数据图片；
  5. 当购物车为空时，“结算 / 删除”按钮是否为灰色不可点击，且总计金额为“0”；
  6. 当购物车有可购买商品时（普通商品或者限购数未满的限购商品），右上角“编辑 / 完成”是否显示且可点击；
  7. 当购物车有可购买商品时（普通商品或者限购数未满的限购商品），总计金额是否计算正确，“结算 / 删除”按钮是否为橙色且可点击；
  8. 当购物车仅有不可购买商品时（限购已满商品），右上角“编辑 / 完成”是否显示且可点击；
  9. 当购物车仅有不可购买商品时（限购已满商品），总计金额是否为“0”，且“结算 / 删除”按钮是否为灰色不可点击；
  10. 当购物车同时有可购买及不可购买商品时，右上角“编辑 / 完成”是否显示且可点击；
  11. 当购物车同时有可购买及不可购买商品时，总计金额是否计算正确（不可购买商品不计入），且“结算 / 删除”按钮是否为橙色且可点击；
  12. 以上情况点击结算跳转结算界面是否会报网络错误；
  13. 下拉刷新时购物车数据是否刷新（各商品剩余数量等）；
  14. 下拉刷新后点击商品是否跳转至最新一期商品详情；
  15. 下拉刷新后点击商品是否跳转到最新一期商品详情界面；
  16. 退出账号登录不同账号，购物车数据是否会冲突；
  17. 限购商品已满限购数量是否显示“限购数已用完”；

（限购相关可看限购模块）

侧滑删除：

  1. 侧滑购物车商品是否显示删除按钮，点击删除按钮是否删除数据；
  2. 侧滑删除数据后，使购物车在各个情况下（有商品，无商品，有可购买商品，同时有可购买不可购买商品等情况），右上角“编辑 / 完成”、总计金额、“结算 / 完成”状态是否正确显示，各状态参考上一点；
  3. 在编辑状态下，是否不可侧滑删除；
  4. 侧滑删除后点击结算是否正确跳转至支付界面，且无网络错误；

编辑删除：

  1. 点击右上角“编辑”界面是否进入编辑状态，“结算”按钮是否置为“删除”；
  2. 删除数据后，使购物车在各个情况下（有商品，无商品，有可购买商品，同时有可购买不可购买商品等情况），右上角“编辑 / 完成”、总计金额、“结算 / 完成”状态是否正确显示，各状态参考上一点；
  3. 点击完成是否重置界面，“删除”按钮置为“结算”等；
  4. 点击删除是否正确删除商品，若商品清空，右上角“编辑 / 完成”按钮是否隐藏，总计金额是否为0；

按钮点击：

  1. 若数量为“1”，“-”号按钮是否为不可点击；
  2. 若数量为最大可购买数，“+”号按钮是否为不可点击；
  3. 最大可购买数：若是普通商品则为剩余人次，若为限购商品则为限购数减去已购买数量；
  4. 点击“-”号按钮，数量是否正确减少，若数量减少为“1”，“-”号按钮是否为不可点击；
  5. 点击“+”号按钮，数量是否正确增加，若数量增加为最大可购买数，“+”号按钮是否为不可点击；
  6. 按钮点击后，总计金额是否有正确刷新；

按钮长按：

  1. 长按“-”号按钮后，商品数量是否置为“0”，且“-”号按钮不可点击；
  2. 长按“+”号按钮后，商品数量是否置为最大可购买数，且“+”号按钮不可点击；
  3. 限购商品长按“+”号按钮后是否正确提示已购数量及限购剩余数量，并将数量置为限购剩余数量；
  4. 长按按钮后，总计金额是否正确刷新；

编辑数量：

  1. 点击数量是否可唤起键盘编辑数量；
  2. 唤起键盘是否挡住正在编辑的文本框；
  3. 输入超过最大可购买数是否会自动置为最大可购买数，且“+”号按钮置为不可点击；
  4. 清空文本框并结束编辑，商品数量是否置为“1”，且“-”号按钮是否置为不可点击，

点击结算：

  1. 点击结算是否跳转至结算界面；
  2. 结算界面金额及商品及商品购买人次是否正确；

“我的”模块

显示：

  1. 若未登录，签到、充值、用户信息等是否隐藏，且显示默认头像，显示点击登录；
  2. 若未登录，点击头像是否跳转至登录界面；
  3. 若已登录是否正确显示用户名、用户头像、福分及余额；
  4. 点击签到是否可正确签到；
  5. 签到成功福分是否响应增加，且“签到”按钮置为“已签到”并不可点击；
  6. 退出账号并重新登录，查看是否可再次签到，且信息是否正确；
  7. 若已登录，点击头像是否跳转至个人资料界面；

个人资料：

  1. 个人资料界面数据是否正确显示；
  2. 点击上传是否可以成功上传头像（相册 & 照相机）；
  3. 上传头像成功，个人资料界面头像是否及时设置，返回至我的界面头像是否也设置完成；
  4. 上传头像失败，头像是否还为默认头像；
  5. 点击地址管理是否进入地址列表界面；

地址管理：

  1. 地址管理界面数据显示是否正确；
  2. 点击右上角“+”号添加地址，输入相关信息提交地址，返回地址列表界面，提交的数据是否展示；
  3. 点击删除按钮，地址信息是否正确删除；
  4. 点击编辑按钮，是否进入编辑界面，编辑地址是否可以正确提交；
  5. 提交编辑后的地址，返回地址列表查看数据是否已置为编辑后的地址；
  6. 点击设为默认地址，返回重新进入查看是否正确设置；

购买记录：

  1. 购买界面商品数据是否正确显示；
  2. ”全部“栏目是否以”正在揭晓“→”尚未揭晓排序“→”已揭晓“排序；
  3. ”进行中“栏目是否显示其应显示数据，且按时间排序；
  4. ”已揭晓“栏目是否显示其应显示数据，且按时间排序；
  5. 尚未揭晓的商品，点击”追加“是否会正确跳转至该期商品详情；
  6. 点击查看号码，是否可以跳转至号码界面；
  7. 包尾购买的商品是否会立即揭晓；

中奖纪录：

我的晒单：

邀请好友：

  1. 邀请好友界面好友数及好友列表是否正确显示；
  2. 购物提成是否正确显示；
  3. 点击邀请好友，是否可分享链接给微信好友或朋友圈；
  4. 打开分享的链接查看是否有数据，且数据是否正确；

联系客服：

  1. 进入联系客服界面，客服电话是否正确显示，且点击手机图标，是否显示拨打电话；

设置：

  1. 点击设置界面的各个条目查看是否有数据，且数据是否正确；
